## 🔷 sitecue UI マイクロインタラクション運用ガイドライン

### 1. 実装の目的とコンセプト
sitecueの「引き算の美学」に基づき、UIのノイズ（常時表示されるテキストや過剰な装飾）を極力排除しつつ、ユーザーの操作（ホバー・クリック）に対して直感的で心地よいフィードバックを提供することを目的とする。
本プロジェクトにおけるアニメーションは、**JSを用いないTailwind CSS（GPUアクセラレーション）による実装**を標準とし、パフォーマンスとUXを両立させる。

### 2. 共通アニメーションボタンの使い分け

新たに作成した3つのステートレス・ラッパーコンポーネントは、用途に応じて以下のように使い分けること。

| コンポーネント | 概要・挙動 | 主なユースケース・適用例 |
| :--- | :--- | :--- |
| **`HoverRevealButton`** | **スライドイン型**<br>通常はアイコンのみ。ホバー時に横からテキストが滑らかに出現する。 | アクションの意味をテキストで補足したい重要な操作。（例：Edit, Delete等） |
| **`HoverSwapButton`** | **スワップ型**<br>ホバー時に縦方向のスライド＆フェードで別のアクションアイコンに入れ替わる。 | 状態変化を伴うワンアクション操作や、関連する代替アクション。（例：Copy ⇔ Checkmark等） |
| **`AnimatedIconButton`** | **バウンス＆フィル型**<br>ホバーで微拡大（`scale-110`）、クリックで微縮小（`scale-95`）。Active時に塗りつぶしアイコンへ遷移。 | ON/OFFの状態（トグル）を持つピン留めや、お気に入り機能。（例：Pin, Favorite等） |

### 3. 実装・運用時の厳守ルール (Do's & Don'ts)

* **レイアウトシフトへの配慮 (Do):**
  `HoverRevealButton` はホバー時に自身の横幅（`max-w`）が変動する。使用する際は、隣接する要素が不自然に押し出されて画面全体がガタつかないよう、親要素（Flexコンテナなど）の余白や幅の設定を適切に行うこと。
* **過剰な装飾の禁止 (Don't):**
  アニメーション（動き）自体が強力な視覚的フィードバックとなるため、ホバー時の「背景色の変化」などは原則として無効化（`hover:bg-transparent`）し、情報の衝突・ノイズを防ぐこと。
* **アイコンのアクセシビリティ (Do):**
  視覚的な装飾目的のSVGアイコン（Lucide React等）を渡す際は、スクリーンリーダーに不要な読み上げをさせないため、必ず **`aria-hidden="true"`** を付与すること。
* **ボタンのデフォルト隙間（Gap）に注意 (Do):**
  ベースとなる `Button` コンポーネントの仕様により予期せぬ余白が生まれる場合がある（今回のアイコン左寄り問題など）。必要に応じて `gap-0` や `px-2` などを明示的に渡し、通常時の美しいセンタリングを担保すること。
