# 外部API連携（AIモデル・サードパーティ）のルール

## 1. モデルIDの厳格な環境変数管理
Geminiやその他の外部APIを利用する際、AI（あなた）はモデル名やバージョンストリングを推測で指定してはいけません。
また、**`.ts`ファイル（ロジック内）にモデル名（例: `gemini-1.5-flash` 等）を直接ハードコード（ベタ書き）することは厳禁**です。

- 開発環境と本番環境での柔軟なモデル切り替えを可能にするため、モデル名は必ず環境変数（例: Honoの `c.env.GEMINI_MODEL_NAME`）から注入するアーキテクチャにすること。
- 未設定時のフォールバックとしてのみ、公式ドキュメントで確認された現在有効な正確なモデルIDをデフォルト値として記述することを許可します（`latest` などの自動追従エイリアスは予期せぬ出力変化を招くため非推奨）。
